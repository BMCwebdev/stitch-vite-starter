# Setup Report: stitch-vite-starter

**Date:** 2026-03-25
**Purpose:** Document how this Vite + React + Stitch template repository was created.

---

## Goal

Create a Vite-based equivalent of the existing `platform-frontend-template` (Next.js) that comes with the Stitch design system pre-configured and ready to use. The template should be lean â€” no routing, no deployment config, no Test Drive â€” just a working Stitch + StyleX + Vite setup that proves the full pipeline works.

## Reference Material

We used three primary sources:

1. **`platform-frontend-template`** (the existing Next.js template) â€” for dependency versions, .npmrc config, prettier/husky/lint-staged setup, and the `copy-stitch-docs.mjs` postinstall script
2. **Stitch README** (`/Development/stitch/README.md`) â€” for understanding Stitch's installation requirements, peer dependencies, and font configuration
3. **StyleX documentation** (`stylexjs.com`) â€” specifically the Vite installation guide and `@stylexjs/unplugin` API docs

## Key Architectural Differences from Next.js Template

| Aspect                       | Next.js Template                                             | Vite Template                                                |
| ---------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **StyleX compiler**          | `@stylexjs/babel-plugin` + `@stylexjs/postcss-plugin`        | `@stylexjs/unplugin` (handles everything)                    |
| **Node modules compilation** | `transpilePackages` in next.config + PostCSS `include` paths | `externalPackages` option on the unplugin                    |
| **CSS injection**            | PostCSS `@stylex` directive in `styles/stylex.css`           | Unplugin appends to CSS bundle automatically                 |
| **Fonts**                    | `next/font/google` with `variable` prop                      | Google Fonts CDN `<link>` in `index.html`                    |
| **Testing**                  | Jest + babel-jest with StyleX babel plugin                   | Vitest with `server.deps.inline` to force StyleX compilation |
| **Routing**                  | Next.js App Router (file-based)                              | None (single page)                                           |
| **Deployment**               | SST/AWS                                                      | None                                                         |
| **Package manager**          | pnpm                                                         | pnpm (same)                                                  |
