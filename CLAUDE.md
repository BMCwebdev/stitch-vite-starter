# Stitch Vite Starter

## Project Overview

This is a Vite + React single-page application template with the **Stitch** design system pre-configured. Stitch is Bonterra's internal design system built on StyleX and React Aria Components.

## Tech Stack

- **Vite 6** — Build tool and dev server
- **React 19** — UI framework
- **TypeScript** — Type-safe development
- **StyleX** — CSS-in-JS styling (via `@stylexjs/unplugin`)
- **Stitch** — Design system (`@bonterratech/stitch-extension` + `@bonterratech/stitch-tokens`)
- **React Aria Components** — Accessible component primitives (peer dependency of Stitch)
- **Vitest** — Testing framework
- **pnpm** — Package manager

## Key Commands

- `pnpm dev` — Start dev server
- `pnpm build` — Type-check and build for production
- `pnpm preview` — Preview production build
- `pnpm test` — Run tests
- `pnpm test:watch` — Run tests in watch mode
- `pnpm lint` — Run ESLint

## Architecture Notes

- **StyleX compilation**: Configured via `@stylexjs/unplugin` in `vite.config.ts`. The `externalPackages` option ensures Stitch's uncompiled StyleX in `node_modules` gets processed at build time.
- **CSS import order**: `stitch-extension.css` is imported before `index.css` in `src/main.tsx`. StyleX-generated CSS is appended to the CSS bundle by the unplugin.
- **Fonts**: Noto Sans and Noto Sans Mono loaded via Google Fonts CDN in `index.html`.
- **No routing**: This is a single-page starter. Add your own router as needed.

## Stitch Documentation

After `pnpm install`, AI-friendly Stitch documentation is available in `stitch-ai-assets/`. This is generated by the postinstall script and gitignored.

## Registry Configuration

The `.npmrc` routes scoped packages to private registries:

- `@bonterratech` → GitHub Packages
- `@fortawesome` / `@awesome.me` → FontAwesome

Auth tokens must be configured in your user-level `~/.npmrc`.
